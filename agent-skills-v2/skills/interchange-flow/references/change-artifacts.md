# Change Artifact Templates

Use these headings when creating an Interchange OpenSpec-like change package.

## proposal.md

```md
# Proposal: <change-id>

## Background

## Goals

## Non-Goals

## Affected Scope

## Source Material

## Risks

## Acceptance Criteria

## Open Questions
```

## context.md

```md
# Context: <change-id>

## Goal

## Must-Read Specs

## Must-Read Docs

## Must-Read Code

## Confirmed Rules

## Constraints

## Verification Commands

## Risks

## Open Questions
```

## design.md

```md
# Design: <change-id>

## Approach

## Interfaces

## Data Flow

## Failure Modes

## Compatibility

## Alternatives Considered
```

## tasks.md

```md
# Tasks: <change-id>

- [ ] Confirm scope and open questions.
- [ ] Implement code changes.
- [ ] Update or add tests.
- [ ] Run verification.
- [ ] Update implementation notes.
- [ ] Prepare archive.
```

## delta-spec.md

```md
# Delta Spec: <change-id>

## ADDED

## MODIFIED

## REMOVED

## Affected Specs

## Questions Needing Confirmation
```

## implementation-notes.md

```md
# Implementation Notes: <change-id>

## Summary

## Files Changed

## Decisions

## Verification

## Follow-Ups
```

## review.md

```md
# Review: <change-id>

## Human Confirmation

## AI Review Notes

## Risks Accepted

## Archive Readiness
```
