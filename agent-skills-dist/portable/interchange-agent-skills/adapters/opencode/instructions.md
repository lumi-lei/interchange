# OpenCode Interchange Instructions

Use the Interchange core Markdown files as reusable workflow instructions:

- `core/roles.md`
- `core/workflow.md`
- `core/prompt-patterns.md`
- `core/confirmation-policy.md`
- `core/openspec-like.md`

Default to standalone drafting and context generation. Connected server use and webhook sending are optional enhancements requiring explicit user instruction and confirmation.

For project changes, follow the OpenSpec-like loop: explore -> propose -> context -> apply -> archive. Create proposal/context/design/tasks/delta-spec artifacts first, then pause for human confirmation before modifying product code.
